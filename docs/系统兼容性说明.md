## 二、运行时依赖

### 发布模式：独立部署（Self-Contained）

两个架构版本均已内置 .NET 6.0 运行时，无需安装任何额外组件。

| 依赖项 | 是否需要单独安装 |
|--------|:---:|
| .NET 6.0 Runtime | ❌ 不需要 |
| ASP.NET Core Runtime | ❌ 不需要 |
| 其他运行时/框架 | ❌ 不需要 |

> **结论：** 根据目标机器架构选择对应目录，复制即可运行。

### 系统级依赖

| 依赖 | 说明 | 缺失时影响 |
|------|------|------------|
| Windows 串口驱动 | 操作系统自带 COM 端口驱动 | 无法枚举/操作串口 |
| 管理员权限 | 安装 Windows 服务时需要 | 无法 install/uninstall |
| TCP 9600 端口 | Web 服务监听端口 | 服务无法访问 |
| 防火墙放行 | 远程访问时需要 | 仅本机可访问 |

## 五、已知限制

| 限制项 | 说明 | 影响 |
|--------|------|------|
| **双版本分发** | x64/x86 需分别发布 | 维护两个 exe |
| **仅 Windows** | 使用 Win32 API (WMI/串口/服务) | 无法跨平台 |
| **单文件体积大** | ~83-89 MB（含完整运行时） | 首次分发较大 |
| **.NET 6.0 EOL** | .NET 6.0 已于 2024-11-12 停止支持 | 不再接收安全补丁 |
| **Win7 需 SP1** | .NET 6.0 最低要求 Win7 SP1 | RTM 版本无法运行 |
| **Win7 x86 需 ESU** | 32 位 Win7 需要扩展安全更新补丁 | 旧版 x86 Win7 可能异常 |
| **需要管理员权限** | 安装/卸载 Windows 服务 | 非管理员用户仅可前台运行 |

## 六、Win7 部署指南

### 前置条件检查

1. 确认系统架构：
   - 64 位：使用 `publish\x64\SerialPortService.exe`
   - 32 位：使用 `publish\x86\SerialPortService.exe`
2. 确认系统为 **Windows 7 SP1** 及以上
3. 安装所有 Windows Update 可用补丁（尤其是 KB2533623）
4. 确认串口驱动正常（设备管理器可看到 COM 端口）

### 部署步骤

```cmd
:: 1. 将对应架构的 publish 文件夹复制到目标机器（如 C:\SerialPortService）
:: 2. 以管理员身份打开 cmd，进入目录
cd C:\SerialPortService

:: 3. 直接运行测试
SerialPortService.exe

:: 4. 确认正常后，安装为 Windows 服务
SerialPortService.exe install
SerialPortService.exe start

:: 5. 验证
SerialPortService.exe status
curl http://localhost:9600/api/health
```

### 防火墙配置（如需远程访问）

```cmd
netsh advfirewall firewall add rule name="SerialPortService" dir=in action=allow protocol=TCP localport=9600
```

## 八、故障排查

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| `不是有效的 Win32 应用程序` | 架构不匹配 | 64 位系统用 x64 版，32 位系统用 x86 版 |
| `无法启动此程序，因为计算机中丢失 api-ms-win-*` | Win7 未安装 SP1 或缺少补丁 | 安装 Win7 SP1 + KB2533623 |
| `拒绝访问` (install 命令) | 未以管理员身份运行 | 右键 → 以管理员身份运行 |
| `端口 9600 已被占用` | 已有其他进程占用 | 修改 appsettings.json 中的端口号 |
| 串口列表为空 | 无串口设备或驱动异常 | 检查设备管理器 |
